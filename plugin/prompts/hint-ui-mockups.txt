 When a plan section describes a UI layout, screen, or component (e.g. a frontend feature, a dashboard widget, a form, a modal), embed a concise inline HTML/CSS mockup to illustrate it — a self-contained <div> with inline styles, placed DIRECTLY in the markdown body (NOT inside backtick code fences, or the HTML will render as raw code instead of a visual). Keep it short and focused on the idea, not production-ready. Inline SVG is also fine for simple icons or small illustrations.
