 When a plan section describes a UI layout, screen, or component (e.g. a frontend feature, a dashboard widget, a form, a modal), embed a concise inline HTML/CSS mockup to illustrate it — a self-contained <div> with inline styles, placed DIRECTLY in the markdown body (NOT inside backtick code fences, or the HTML will render as raw code instead of a visual). Do NOT draw UI with ASCII-art boxes (│ ├ └ ─ etc.) — use real HTML so scratchtml renders it as a visual. For example: <div style="border:1px solid #ccc;border-radius:8px;padding:12px;max-width:320px;font-family:sans-serif"><div style="font-weight:600;margin-bottom:8px">Card title</div><button style="background:#2563eb;color:#fff;border:0;border-radius:6px;padding:6px 12px">Action</button></div>. Keep it short and focused on the idea, not production-ready. Inline SVG is fine for simple icons or small illustrations. If a frontend-design skill or plugin is available in this session, use it to author higher-fidelity mockups.
